Setting Up Okta API Token

Creating a Service Account

  1. Access your Okta tenant administration dashboard
  2. Create a dedicated service account or synchronize from your directory service
  3. Ensure the account is set to ACTIVE status with no expiration date
  4. Store the service account credentials securely in your password vault

Configuring Administrative Roles

  1. Log in to the Okta admin dashboard with Super Administrator credentials
  2. Navigate to the newly created service account
  3. Select the "Admin Roles" tab
  4. Click "Add Individual Assignments"
  5. Assign the following roles:
    • Help Desk Administrator
    • Super Administrator (temporary)
  6. Verify both role assignments are active

Generating the API Token

  1. Open a new private browsing session
  2. Log in using the service account credentials
  3. Navigate to Security > API > Tokens
  4. Select "Create Token"
  5. Configure the token:
    • Name: Enter a descriptive name (e.g., "fctr-id-portal")
    • IP Address Range:
      • During trial period: Select "Any IP"
      • Production: Use FCTR-provided IP address
  6. Copy and securely store the generated token

    7. ⚠️ CRITICAL SECURITY ACTION REQUIRED

    Stop here and remove the Super Administrator role assigned to the service account before proceeding further

SAML Configuration

Creating SAML Integration

  1. Access Okta admin console with Super Administrator credentials
  2. Navigate to Applications > Create App Integration > SAML 2.0
  3. Configure basic settings:
    • Application name: "FCTR Identity Portal"
    • Upload the FCTR logo image. Download logo here

SAML Settings Configuration

Required Endpoints

  • Single Sign-on URL: https://<tenant>.fctr.io
  • Audience URI (SP Entity ID): <tenant>-fctr
  • Name ID format: Unspecified (recommended)

Additional Attributes

Attribute Name Okta Expression
email user.email
firstName user.firstName
lastName user.lastName

Required Groups for FCTR portal

A minimum of three groups are required to be created in your Okta tenant for FCTR portal access role-based access

The three roles in FCTR are:

  • Super admin
  • HD admin
  • HD user

Group Attributes (options)

Attribute Name Filter Okta Expression
groups Starts With - OR - contains (Recommended) GROUP-Prefix
groups Matches regex (Not Recommended) .*
